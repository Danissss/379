Objectives

This project give me the experience of developing multithreaded programs that require thread
synchronization and deadlock prevention. In order to protect the critical section during the
multithreading, using the semaphores our mutex implementation could efficiently prevent one
section (e.g. critical section) to be accessed by multiple threads at the same time, which prevent
the race condition and deadlock during multithreading.

Design Overview

  • Multithread creation is done by pthread_create and pthread_join from pthread library from linux.
    The preventation of race condition is done by calling the API of mutex, such as mutex_lock and
    mutex_unlock.
  • Program may have up to 10 resource types, and 25 tasks. A task or a resource type name has at most 32 characters.
    Each white space between fields is composed of one, or more, space character(s), and there here is no white space
    around the ’:’ field separator. This composes of inputFile.
  • Each task is defined by taking certain amount of resources and run as a task thread. If the resource is not available,
    • then the task will wait until the resource is available, and finish its tasks.
  • There is a monitor thread that will record the current status of all task threads, which in "WAIT", "IDLE" and "RUN"
    categories.
  • When all task threads finish running, then the monitor thread will exit too. The final report for the whole process
    will be printed on the screen.

Project status

You can successfully create numbers of task thread and the monitor thread.
The idle and busy time is defined by the line inside the input File. You can set the monitor time, which will print
the current status of all task thread per millisecond. You can also define the number of times that each task need
to run (i.e. define the number of iterations).

The difficult part of this assignment is:

  • 1)It takes time to understand the mutex library and pthread library, as well as how to use them correctly.
  • 2)Parsing information is very complex; regular expression could be a great helper for future implementation.
