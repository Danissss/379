Objectives

This project give me experience of understanding how the utilize fifo to transmit message or signal from one process to different one. It also points out the importance of utilizing the I/O multiplexing for nonblocking I/O, which one program can recive different I/O from different resources. In our case, we need to make the program to receive fifo as well as user input (e.g. list and exit). The assignment also introduces the concept of controller (i.e. router, server) and switches (i.e. clients) that how they communicate through fifo; although, there are multiply methods for communication, but the concept of communication are similar.

Design Overview

  • There is only one controller and 7 switches exist, which controller will listen all 7 switches and be able to recieve the message (e.g. requesting for open port) from swithes and also need to send back message (e.g. allow for connection). My program can indeed create a controller and numbers of switches that can communicate with each other through fifo. My switch will send open request to controller; and controller will send switches the allowance message for confirmation.
  • My program also can recieve multi-I/O from different sources without interruption.
  • The message that send between controller and switch are construct as string. After controller/switch gets the message, it will parse the string based on pre-defined rules. In the later implementation, the c struct should be used for construct the message.
  • My program also keep updating the current statistics about the packet sent and recieved; as well as the number of package got and lost during processing the packet. The "list" command inside the controller and switch program can list the current statistics of the package information. "Exit" command will terminate the program

Project status

You can successfully create controller and swithes by using correct function call.
Each time a new swith starts, it will sends message to controller; and then, the controller will sends confirm message back.
"list" command in each process will print out the current statistics of the package information; and "exit" command will terminate the program
The difficult part of this assignment is
